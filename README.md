# 义务维修申请小程序用户端

基于 Vue 开发，用于用户申请义务维修。

## 项目目标

实现一个完整的用户侧体验，包含注册登录、首页、消息列表、维修记录、用户中心等功能。

## 功能概要

- 注册 / 登录
  - 输入项：姓名、学号、手机号码
  - 使用手机验证码登录；未注册用户自动注册
  - 登录流程需展示并要求用户同意“维修协议”选项

- 首页：
  - 顶部新闻轮播卡：图片 + 文字，支持轮播
  - 公告卡（高度较低，分两行）：例如“第X次义务维修活动将于本周末在XX进行，时间 XX年XX月XX日 XX:XX-XX:XX”
  - 并列两按钮卡：左“申请义务维修”，右“问维修智能体”

- 消息页：
  - 消息列表，按时间倒序展示通知/对话条目

- 维修记录：
  - 卡片式记录列表，按日期分组、从上到下排列，样式类似电商订单记录页

- 我的
  - 显示用户头像、昵称、姓名、学号
  - 可进入的选项：维修协议、我的申请、维修记录（跳转到维修记录页）、我的信息（子页面：修改头像/昵称/姓名/学号/手机号）、关于我们、意见反馈

## 开发与运行

  通过以下命令安装并启动demo演示：

  ```bash
  npm install
  npm run dev
  ```

  ## 原始 AI 提示词

  以下是用于生成本项目的原始AI提示词。

  > 设计一个维修全流程小程序的用户侧页面。
  > 需要包含如下页面。
  > 1.用户注册登录页：需要填写姓名、学号、手机号码，并接收手机验证码实现登录（并提示，未注册的用户会自动注册，并附有“同意维修协议”的选项）
  > 2.内容页面：在完成注册登录后会进入内容页，以下a、b、c、d四个页面在底部控制栏从左到右显示按钮。
  > -a：首页：需要包含以下内容，以卡片的形式展示：顶部，有一个新闻框，用来展示最近发生的新闻之类的，图片配合文字，要求做到内容轮播；下方，有一个框，这个要高度低一点，内容是，第X次义务维修活动将于本周末在XX进行，时间XX年XX月XX日 XX:XX-XX:XX（分成两行）；在这个的下方，放置左右并列的两个框，左边是“申请义务维修”，右边是“问维修智能体”。
  > -b：消息页面：这个页面做一个消息列表。
  > -c：维修记录：这个页面要做成一个一个卡片式的，从上到下根据日期依次排布，类似于淘宝、美团那种订单记录页面。
  > -d：我的：用户页面，就是很经典的那种用户信息页面，需要展示：用户的头像、昵称、姓名、学号，然后下面有一些可以点进去的设置或查看项，比如：维修协议、我的申请、维修记录（跳转到c页面）、我的信息（点开之后子页面：可以修改头像、昵称、姓名、学号、手机号）、关于我们、意见反馈。
